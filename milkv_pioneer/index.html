RVV benchmark Milk-V Pioneer (SG2042, C910)

The Milk-V Pioneer is based on SOPHON SG2042 with 64 XuanTie C910 cores. It has a frequency 2GHz and implements rv64gc and the XTheadVector extension, based on the 0.7.1 version of the vector extension supporting SEW=8/16/32/64, single, half and double precision floating point instructions, with a VLEN of 128 bits.
Note: The core (C910 + XTheadVector) was originally marketed as C920, but XuanTie retroactively changed their marketing to refer to the XTheadVector cores as C910. The official C920 will be a modified C910 with RVV 1.0 that is yet to release.

Performance observations

Note that all of these only apply to this processor and aren't set in stone.

  • Maximizing LMUL is a good strategy for computation, it looks like LMUL=4 is better when memory bottlenecked (I'd recommend aiming for LMUL=4 as a default).
  • The vl doesn't impact performance, so a vl=1 and a vl=VLMAX instruction with LMUL=8 will take the same amount of time to execute.
  • The fault-only-first load instructions are quite slow compared to the regular load instructions, so one should try to page align the loads by hand when possible. (see strlen)
  • When a gather operation doesn't need to cross register bounderies, multiple LMUL=1 or LMUL=2 gathers are faster. (see byteswap)
  • The results from LUT4 are confusing, my best bet is that the added vsetvli cause the slowdown, otherwise rvv_gather_m1 shouldn't be faster than rvv_m1_gather_m2.
  • When not reading and doing much computation, SWAR implementations can be faster than RVV for large input sizes. (see memcpy)
  • Manual tail handling and unrolling doesn't seem to be worth it in these benchmarks.
  • Aligning the destination pointer in a streaming workload, like memcpy can make the runtime slightly faster for medium large sizes, but slows it down for smaller sizes.
  • Uni-stride segmented stores and strided stores, are way slower than uni-stride stores. (see ascii to utf16/utf32)
  • Using the mask operand seems to slow down arithmetic operations with LMUL smaller than 8 in the measurement. But this is likely because of the added dependency to the previous instruction, and won't be as visible in practice.
